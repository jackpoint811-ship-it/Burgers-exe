# Flujo public-order

## Objetivo

El flujo público debe permitir pedir burgers de forma rápida, clara y mobile-first.

## Principios

- Navegación por pasos clara.
- CTA visible.
- Sin overflow horizontal en móvil.
- Textos simples.
- Estados loading, success y error claros.
- Mantener estética Burgers.exe.

## QA mínimo

- Probar en 320px.
- Probar en 390px.
- Probar desktop.
- Validar pedido completo.
- Validar forma de pago.
- Validar mensajes de error.
