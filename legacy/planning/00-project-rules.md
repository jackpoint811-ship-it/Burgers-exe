# 00 — Project Rules (Fase 0)

## Reglas obligatorias
1. No implementar funcionalidad nueva durante Fase 0.
2. No crear backend nuevo durante Fase 0.
3. No crear UI nueva durante Fase 0.
4. No modificar integraciones con Google Sheets durante Fase 0.
5. Mantener todo legado previo dentro de `legacy/`.
6. `legacy/` no se elimina sin autorización explícita del usuario.
7. Codex no avanza de fase sin autorización explícita del usuario.
8. Toda fase debe cerrar con bitácora antes de avanzar.
9. Enfoque mobile-first: teléfono vertical primero, tablet después.
10. Todo cambio de Fase 0 debe ser documental y trazable.

## Stack y límites
- Stack permitido: Google Sheets + Google Apps Script Web App + HTML/CSS/JS embebido.
- Sin librerías externas salvo decisión explícita posterior.
- Prohibido usar servicios externos en cualquier fase salvo aprobación explícita del proyecto.
